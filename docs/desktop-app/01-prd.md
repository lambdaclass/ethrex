# PRD: Tokamak Desktop App (기획서) v2

## 1. 제품 비전

카카오톡과 같은 친숙한 데스크탑 메신저 형태의 애플리케이션.
AI(L2 Pilot)와 대화하며 **여러 개의 Tokamak L2 네트워크를 생성·운영·관리**하고,
**오픈 L2 생태계**를 통해 다른 사람의 L2를 탐색·연결·사용할 수 있는 올인원 플랫폼.

## 2. 핵심 가치

| 가치 | 설명 |
|------|------|
| **접근성** | 설치형 데스크탑 앱으로 누구나 쉽게 L2를 만들고 운영 |
| **AI-First** | "L2 Pilot" AI와 채팅으로 모든 기능을 수행 (명령어 학습 불필요) |
| **멀티 L2** | 한 앱에서 여러 개의 L2를 생성하고 동시에 관리 |
| **오픈 생태계** | 카카오 오픈채팅처럼, L2를 공개하고 서로 연결하는 "오픈 L2" |
| **AI 상호운용** | AI 에이전트가 서로의 L2를 탐색하고 사용할 수 있는 가이드 시스템 |

## 3. 타겟 사용자

| 사용자 유형 | 니즈 |
|------------|------|
| L2 운영자 | 여러 L2를 생성·배포·운영, 서비스를 공개 |
| 개발자 | L2 위에서 dApp 개발, 다른 L2 API 연동 |
| 탐색자 | 다양한 오픈 L2를 둘러보고 서비스 이용 |
| AI 에이전트 | L2의 가이드를 읽고 자율적으로 기능 수행 |

## 4. 앱 구조 (카카오톡 비유)

카카오톡의 구조를 L2 생태계에 그대로 대응시킨다.

| 카카오톡 | Tokamak Desktop | 설명 |
|----------|----------------|------|
| 친구 목록 | **내 L2** | 내가 만든 L2 목록, 생성·관리 |
| 채팅 | **L2 Pilot (AI 채팅)** | AI와 대화하며 앱 제어 |
| 오픈채팅 | **오픈 L2** | 다른 사람의 L2를 탐색, 해시태그 필터링 |
| 톡스토어/더보기 | **대시보드 & 지갑** | 대시보드, TON 충전, 설정 |

## 5. 주요 기능

### 5.1 내 L2 관리 (My L2s)

카카오톡의 "친구 목록"에 해당. 내가 만든 L2들을 한눈에 보고 관리한다.

#### L2 목록
- 내 L2들의 리스트 (이름, 상태, Chain ID, 아이콘)
- 각 L2의 온라인/오프라인 상태 표시
- 선택하면 해당 L2의 상세 관리 화면으로 이동

#### L2 생성 위자드 ("+ 새 L2 만들기")
| 설정 항목 | 설명 | 예시 |
|-----------|------|------|
| L2 이름 | 체인 이름 | "My DEX Chain" |
| Chain ID | 고유 체인 식별자 | 17001 |
| 네이티브 토큰 | 가스비에 사용할 토큰 | TON |
| L1 연결 | 연결할 L1 RPC 엔드포인트 | http://localhost:8545 |
| 시퀀서 모드 | 단독/공유 시퀀서 | 단독 |
| Prover 설정 | ZK 증명 방식 | SP1 |
| 공개 여부 | 오픈 L2로 공개할지 | 비공개 |
| 해시태그 | 검색용 태그 | #DeFi #DEX #AMM |
| 설명 | L2 소개문 | "Low-fee DEX on Tokamak" |

#### L2 상세 관리 (선택된 L2)
- **노드 제어**: 클라이언트/시퀀서/프로버 시작·중지·재시작
- **로그 뷰어**: 실시간 stdout/stderr 스트리밍
- **설정 편집**: 포트, 가스비, 배치 간격 등
- **대시보드 탭**: 해당 L2의 웹 대시보드 (WebView)
- **TON 잔액**: 이 L2의 TON 잔액 및 입출금
- **공개 설정**: 오픈 L2로 등록/해제, 해시태그 편집

### 5.2 L2 Pilot (AI 채팅)

카카오톡의 "채팅"에 해당. AI와 대화하며 앱의 모든 기능을 제어한다.

#### AI 프로바이더 로그인
| 프로바이더 | 인증 방식 | 모델 예시 |
|-----------|----------|----------|
| Claude (Anthropic) | API 키 | Opus 4, Sonnet 4 |
| GPT (OpenAI) | API 키 | GPT-4o, o1 |
| Gemini (Google) | API 키 | Gemini Pro, Ultra |

- API 키는 OS Keychain에 안전 저장
- 설정에서 프로바이더/모델 변경
- 여러 프로바이더를 등록해두고 전환 가능

#### 채팅 UI (카카오톡 스타일)
```
+--------------------------------------------------+
|  🤖 L2 Pilot (Claude Opus)          [모델 선택 ▼] |
|--------------------------------------------------|
|                                                  |
|     Pilot: 안녕하세요! Tokamak L2 관리를         |
|            도와드리겠습니다.                       |
|                                                  |
|  나: 내 DEX Chain 노드 상태 확인해줘              |
|                                                  |
|     Pilot: DEX Chain (Chain 17001) 상태:         |
|            ✅ 시퀀서: Running (PID 4521)          |
|            ✅ 프로버: Running (PID 4522)          |
|            📊 블록 높이: #45,231                  |
|            📦 대기 배치: 3건                      |
|                                                  |
|  나: 오픈 L2에서 NFT 마켓 있는 곳 찾아줘          |
|                                                  |
|     Pilot: #NFT 태그가 있는 오픈 L2를 찾았습니다: |
|            1. NFT Marketplace L2 (856명)          |
|            2. Art Chain (234명)                   |
|            연결해볼까요?                           |
|                                                  |
|--------------------------------------------------|
|  [메시지를 입력하세요...]                  [전송]  |
+--------------------------------------------------+
```

#### AI가 수행할 수 있는 작업
| 카테고리 | 작업 | 예시 대화 |
|----------|------|----------|
| L2 생성 | 새 L2 생성 | "DEX용 L2 하나 만들어줘" |
| 노드 제어 | 시작/중지/재시작 | "DEX Chain 프로버 재시작해줘" |
| 상태 확인 | 노드/배치/증명 상태 | "현재 배치 상태가 어때?" |
| 설정 변경 | 가스비/포트 등 | "가스비 0.001 TON으로 바꿔줘" |
| 대시보드 | 탭 열기/전환 | "DEX Chain 대시보드 보여줘" |
| 지갑 | 잔액/입출금 | "L2에 10 TON 입금해줘" |
| 오픈 L2 | 검색/연결 | "Bridge 서비스 찾아줘" |
| AI 가이드 | 다른 L2 사용법 | "저 DEX에서 스왑하는 법 알려줘" |

#### AI 가이드 시스템
AI가 앱의 기능을 정확히 이해하고 실행할 수 있도록:
- 모든 기능에 Tool Use (Function Calling) 정의
- 시스템 프롬프트에 Tokamak L2 도메인 지식 포함
- 민감 작업(중지, 설정변경, 출금) 전 사용자 확인 팝업
- 오픈 L2의 AI 가이드 문서를 읽고 다른 L2 기능 수행

### 5.3 오픈 L2 (Open L2 Marketplace)

카카오톡의 "오픈채팅"에 해당. 다른 사람의 L2를 탐색하고 연결한다.

#### 오픈 L2 목록
```
+--------------------------------------------------+
|  오픈 L2                          [+ 내 L2 등록]  |
|--------------------------------------------------|
|  🔍 L2 이름, 해시태그로 검색...                    |
|--------------------------------------------------|
|  [전체] [#DeFi] [#NFT] [#Gaming] [#Bridge] [#AI] |
|--------------------------------------------------|
|  🔄 Tokamak DEX        1,234명  ●  2m ago        |
|     Decentralized exchange with AMM pools...      |
|     #DeFi #DEX #AMM                              |
|--------------------------------------------------|
|  🎨 NFT Marketplace     856명   ●  5m ago        |
|     NFT minting and trading platform...           |
|     #NFT #Art #Marketplace                        |
|--------------------------------------------------|
|  🤖 AI Agent Hub       1,678명  ●  15s ago       |
|     L2 for AI agent operations...                 |
|     #AI #Agent #Automation                        |
+--------------------------------------------------+
```

#### 해시태그 시스템
- L2 등록 시 해시태그 부여 (#DeFi, #NFT, #Gaming, #Bridge, #AI, #Social 등)
- 해시태그로 필터링, 텍스트로 검색
- 인기 해시태그 자동 정렬

#### L2 상세 페이지
- L2 이름, 아이콘, 운영자, 설명
- 해시태그, 사용자 수, 온라인 상태
- 연결 정보 (Chain ID, RPC URL)
- **"연결하기"** → 내 앱에 이 L2 추가
- **"대시보드 추가"** → 대시보드 탭에 추가
- **"AI에게 물어보기"** → L2 Pilot에게 이 L2 사용법 질문

#### AI 가이드 표준 포맷
각 오픈 L2는 AI가 읽을 수 있는 가이드를 제공한다:
```yaml
service:
  name: "My DEX"
  capabilities:
    - name: "swap_tokens"
      description: "토큰 스왑"
      endpoint: "/api/swap"
      parameters: [from_token, to_token, amount]
  usage_examples:
    - "TON을 USDT로 바꾸고 싶어" → get_price → confirm → swap
  fees:
    gas_token: "TON"
    estimated_gas: "0.001 TON"
```
→ AI가 이 가이드를 읽고 사용자 대신 해당 L2의 기능을 수행할 수 있다.

### 5.4 대시보드 (Dashboard)

선택된 L2의 웹 대시보드를 WebView 탭으로 표시한다.

| 탭 | 내용 | 기본 URL |
|----|------|----------|
| L1 상태 | L1 체인 블록, 트랜잭션 | `http://localhost:3010` |
| L2 상태 | L2 체인 블록, 배치, 증명 | `http://localhost:3011` |
| Explorer | 블록 익스플로러 | `http://localhost:3012` |
| Prover | 증명 생성 상태, 비용 | `http://localhost:3013` |
| Metrics | Grafana/Prometheus | `http://localhost:3000` |

- 각 L2마다 대시보드 URL이 다를 수 있음 (L2 설정에서 지정)
- 탭 추가/제거/URL 커스터마이징 가능
- 오픈 L2의 대시보드도 탭으로 추가 가능

### 5.5 TON 지갑 (Wallet)

#### 내 지갑 등록
- 지갑 주소 직접 입력 또는 MetaMask 등 외부 지갑 연동
- 여러 지갑 주소를 등록하고 관리
- 기본 지갑 설정

#### L2별 잔액 관리
각 L2마다 네이티브 토큰(TON)이 별도로 존재한다.
```
┌─────────────────────────────────────────────┐
│  내 지갑: 0x1234...abcd                      │
│─────────────────────────────────────────────│
│  L1 (Ethereum)              125.0 TON       │
│  ├─ DEX Chain (17001)        50.0 TON       │
│  ├─ NFT Chain (17002)        20.0 TON       │
│  └─ Test Chain (17003)        5.0 TON       │
│─────────────────────────────────────────────│
│  AI 지갑: 0xAI12...ef56                      │
│  ├─ DEX Chain (17001)         1.0 TON       │
│  └─ NFT Chain (17002)         0.5 TON       │
└─────────────────────────────────────────────┘
```

| 기능 | 설명 |
|------|------|
| 지갑 등록 | 지갑 주소 입력/외부 지갑 연동, 다중 지갑 관리 |
| 잔액 조회 | L1 + 각 L2별 TON 잔액을 한눈에 표시 |
| Deposit | L1 → 특정 L2로 TON 입금 (L2 선택) |
| Withdraw | 특정 L2 → L1으로 TON 출금 |
| AI 지갑 | AI 에이전트 전용 지갑, L2별로 충전 |
| 트랜잭션 히스토리 | L2별 입출금 내역 |

- AI가 L2에서 작업하려면 해당 L2의 TON이 필요 → AI 지갑에 L2별로 충전
- 여러 L2의 잔액을 트리 구조로 한눈에 확인

## 7. 데이터 흐름

### AI 채팅 → 앱 제어 흐름
```
사용자 메시지
    ↓
L2 Pilot (AI)
    ↓ Tool Use
앱 내부 기능 실행
    ↓ 결과 반환
AI 응답 생성
    ↓
채팅 UI 표시
```

### 오픈 L2 → AI 연동 흐름
```
사용자: "저 DEX에서 토큰 스왑해줘"
    ↓
L2 Pilot: 오픈 L2 레지스트리에서 DEX 가이드 로드
    ↓
L2 Pilot: 가이드에 따라 get_price 호출
    ↓
L2 Pilot: "현재 1 TON = 2.5 USDT입니다. 스왑할까요?"
    ↓
사용자: "응"
    ↓
L2 Pilot: swap_tokens 호출 (AI 지갑에서 가스비 지불)
    ↓
L2 Pilot: "스왑 완료! TX: 0x1234..."
```

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 플랫폼 | macOS, Windows, Linux |
| 프레임워크 | Tauri 2.x (Rust) + React + TypeScript |
| 설치 크기 | < 100MB (런타임 제외) |
| 메모리 | < 200MB (앱 자체, 노드 프로세스 별도) |
| 다국어 | 한국어, English, 日本語 |
| 보안 | API 키 OS Keychain 저장, IPC allowlist, 민감작업 확인 |
| 오프라인 | 노드 제어는 오프라인, AI 채팅은 온라인 필요 |
| 업데이트 | 자동 업데이트 (Tauri updater) |
