# EIP-8272: Recent Roots for Frame Transactions

> **Spec target:** ethereum/EIPs `EIPS/eip-8272.md` (Draft, Standards Track: Core; requires EIP-7843, EIP-8141). Read against `eips.ethereum.org` master on 2026-06-24. Pin the exact spec commit once it is frozen.

Implementation notes for EIP-8272 in ethrex. Builds on EIP-8141 frame transactions and EIP-7843 `SLOTNUM` (both already present); activates under `Fork::Hegota`.

## Overview

EIP-8272 lets a frame transaction declare verified **recent roots** — `(source_id, slot, root)` tuples — in its signed envelope, so validation code (e.g. privacy proofs) can reference a recent commitment without reading another account's mutable storage. References are checked for validity at block execution against the `RECENT_ROOT_ADDRESS` predeploy.

## Implemented (this branch)

| Piece | Detail |
|-------|--------|
| Opcode `RECENTROOTREFLOAD` | `0xB5` (see divergence). Stack `[field, index]` (field on top); `field` 0→source_id, 1→slot, 2→root; gas 3; halts if `index ≥ len` or `field > 2`; reads only the envelope. |
| Envelope field | `recent_root_references: Vec<RecentRootReference>` — last RLP field (the **10th** ethrex field, after `blob_versioned_hashes`; the spec calls it the 9th because it omits ethrex's `signatures`). `RecentRootReference { source_id: H256, slot: u64, root: H256 }`, RLP `[source_id, slot, root]`, rkyv-derived. Covered by `compute_sig_hash` (not elided in VERIFY frames). |
| Static validation | `len ≤ FRAME_TX_MAX_RECENT_ROOT_REFERENCES (16)`; 3-tuple shape + 32-byte `source_id`/`root` + `slot < 2**64` enforced by the type/RLP. |
| TXPARAM `0x0F` | `len(recent_root_references)` (see spec-bug note). |
| `RECENT_ROOT_ADDRESS` | predeploy at `0x…8272` (`crates/vm/system_contracts.rs`). |
| Gas | `RECENTROOTREFLOAD = 3`; `RECENT_ROOT_REFERENCE_ADDRESS_GAS = ACCESS_LIST_ADDRESS_COST`; `RECENT_ROOT_REFERENCE_GAS = ACCESS_LIST_STORAGE_KEY_COST + 2*30 + 7*6`. Both the intrinsic reference gas **and** the EIP-7623 calldata cost over `rlp(recent_root_references)` are added to `FrameTransaction::total_gas_limit` (the references travel in the frame-tx envelope, so their RLP bytes are charged at the 4/16-per-byte calldata rate like `frames`/`signatures`). |

## Divergences from the draft spec

1. **`RECENTROOTREFLOAD` at `0xB5`, not `0xB4`.** The spec's Constants table assigns `0xB4`, which collides with EIP-8141's shipped `SIGPARAM = 0xB4`. ethrex uses the next free byte `0xB5`. Flag upstream.
2. **TXPARAM index `0x0F`** (not `0x0D`). The spec is internally inconsistent — its Constants table says `0x0F`, its summary table says `0x0D`. ethrex uses `0x0F` (Constants table is authoritative). An upstream fix is warranted.
3. **`RECENT_ROOT_ADDRESS = 0x…8272`** — the spec leaves it `TBD`; ethrex uses the `0x…<EIP>` convention.
4. **`RECENT_ROOT_CODE` handled natively** — the spec leaves the bytecode `TBD`; rather than invent bytecode, ethrex handles the 64-byte write (`salt ‖ root`, value 0, static-revert, direct-call-only) as a VM-level special case (see below).

## ⚠ Remaining / flagged for devnet validation

**Scope decision:** this branch ships the EIP-8272 *core* — the envelope field, the `RECENTROOTREFLOAD` opcode, static validation, the per-reference intrinsic gas, and the predeploy address. The consensus-affecting pieces below (reference-validity check, native predeploy write, access-list warming, pre-fork eviction) are **deliberately deferred, not overlooked**: they are the consensus core (analogous to EIP-8250's journaling boundary) and need the multi-client Hegotá devnet to validate. They are tracked here rather than implemented blind.

- **Reference-validity check at block execution:** before the frame loop, for each reference require `1 ≤ current_slot − slot ≤ 8191` (`current_slot` from EIP-7843 `env.slot_number`, never `block.timestamp`), recompute `entry_hash`/`storage_key` (keccak with the `RECENT_ROOT_ENTRY`/`RECENT_ROOT_STORAGE` domains), and assert `RECENT_ROOT_ADDRESS[storage_key] == entry_hash`; an invalid reference invalidates the whole tx (no frames execute). Plus access-list warming of the address + storage keys. **Not yet wired** — this is the consensus core, analogous to EIP-8250's journaling boundary, and needs the devnet + cross-client interop to validate.

  **⚠ Interim safety gate (until the check lands):** because no validity check exists, `RECENTROOTREFLOAD` would otherwise serve *unverified* roots straight from the signed envelope — a soundness hole (a privacy/assertion proof could treat forged bytes as a committed root). To avoid exposing that unsound primitive, `execute_frame_tx` **rejects any transaction carrying a non-empty `recent_root_references` list** (`vm.rs`, before the frame loop). This disables the feature rather than running it unsoundly; replace the gate with the validity check above when it is implemented. (Static validation still accepts the count-bounded list so the envelope/RLP machinery stays testable; the gate lives at execution, where the real check belongs.)
- **Native `RECENT_ROOT_ADDRESS` write** (install + interception): the predeploy account must be created at Hegota (nonce 1, empty code) and writes to it intercepted natively. Address + predeploy entry are in place; the install + write interception are pending.
- **Mempool/builder policy:** reject refs with `slot ≥ current_slot`, evict at `current_slot − slot ≥ 8192`, recheck on head change — local policy, deferred.
- Full VM-execution integration tests.
