# EIP-8272: Recent Roots for Frame Transactions

> **Spec target:** ethereum/EIPs `EIPS/eip-8272.md` (Draft, Standards Track: Core; requires EIP-7843, EIP-8141). Read against `eips.ethereum.org` master on 2026-06-24. Pin the exact commit per change task `0.4` once frozen.

Implementation notes for EIP-8272 in ethrex. Builds on EIP-8141 frame transactions and EIP-7843 `SLOTNUM` (both already present); activates under `Fork::Hegota`.

## Overview

EIP-8272 lets a frame transaction declare verified **recent roots** ‚Äî `(source_id, slot, root)` tuples ‚Äî in its signed envelope, so validation code (e.g. privacy proofs) can reference a recent commitment without reading another account's mutable storage. References are checked for validity at block execution against the `RECENT_ROOT_ADDRESS` predeploy.

## Divergences from the draft spec

1. **`RECENTROOTREFLOAD` at `0xB5`, not `0xB4`.** The spec's Constants table assigns `0xB4`, which collides with EIP-8141's shipped `SIGPARAM = 0xB4`. ethrex uses the next free byte `0xB5`. Flag upstream.
2. **TXPARAM index `0x0F`** (not `0x0D`). The spec is internally inconsistent ‚Äî its Constants table says `0x0F`, its summary table says `0x0D`. ethrex uses `0x0F` (Constants table is authoritative). Upstream fix PR drafted (change task `0.2`/`E.1`).
3. **`RECENT_ROOT_ADDRESS = 0x‚Ä¶8272`** ‚Äî the spec leaves it `TBD`; ethrex uses the `0x‚Ä¶<EIP>` convention.
4. **`RECENT_ROOT_CODE` handled natively** ‚Äî the spec leaves the bytecode `TBD`; rather than invent bytecode, ethrex handles the 64-byte write (`salt ‚Äñ root`, value 0, static-revert, direct-call-only) as a VM-level special case (see below).

## ‚öÝ Remaining / flagged for devnet validation

- **Reference-validity check at block execution** (B.6): before the frame loop, for each reference require `1 ‚â§ current_slot ‚àí slot ‚â§ 8191` (`current_slot` from EIP-7843 `env.slot_number`, never `block.timestamp`), recompute `entry_hash`/`storage_key` (keccak with the `RECENT_ROOT_ENTRY`/`RECENT_ROOT_STORAGE` domains), and assert `RECENT_ROOT_ADDRESS[storage_key] == entry_hash`; an invalid reference invalidates the whole tx (no frames execute). Plus access-list warming of the address + storage keys. **Not yet wired** ‚Äî this is the consensus core, analogous to EIP-8250's journaling boundary, and needs the devnet + cross-client interop to validate.
- **Native `RECENT_ROOT_ADDRESS` write** (B.4 install + interception): the predeploy account must be created at Hegota (nonce 1, empty code) and writes to it intercepted natively. Address + predeploy entry are in place; the install + write interception are pending.
- **Mempool/builder policy** (B.7): reject refs with `slot ‚â• current_slot`, evict at `current_slot ‚àí slot ‚â• 8192`, recheck on head change ‚Äî local policy, deferred.
- Full VM-execution integration tests (B.8).
